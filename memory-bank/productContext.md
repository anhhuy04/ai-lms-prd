# Product Context

## Why AI LMS Exists

### Problem Statement
Traditional assignment handling in educational institutions is broken:
- **Teachers waste time** grading repetitive assignments manually
- **Feedback is delayed** - students submit, wait days/weeks for grading
- **Feedback lacks depth** - time constraints limit personalized insights
- **Learning patterns invisible** - hard to identify struggling students early
- **One-size-fits-all approach** - no adaptive guidance based on individual needs
- **Manual workflows** - copy-pasting, organizing submissions via email/LMS is tedious

### Solution
AI LMS automates and intelligently augments the entire assignment workflow, enabling:
- **Instant feedback** - AI grades within seconds of submission
- **Deep insights** - AI identifies error patterns and generates personalized recommendations
- **Teacher empowerment** - focuses teacher time on high-value activities (feedback refinement, intervention)
- **Student agency** - see progress in real-time, understand mistakes immediately
- **Scalability** - one teacher can now effectively serve more students without burnout

## Target User Personas

### Persona 1: Teacher (Ms. Chen)
- **Role:** High school Mathematics teacher
- **Pain Points:** Spends 3+ hours grading 30 student assignments; repeats feedback across similar mistakes
- **Desired Outcome:** Quick grading, detailed feedback, ability to identify struggling students
- **Behavior:** Creates detailed rubrics; wants analytics to guide interventions
- **Tech Level:** Moderate - comfortable with mobile apps, wants intuitive UI

### Persona 2: Student (Ahmad)
- **Role:** 15-year-old student
- **Pain Points:** Doesn't know if answer is correct until teacher grades (days later); unclear why answers are wrong
- **Desired Outcome:** Immediate feedback, understand mistakes, improve skills quickly
- **Behavior:** Wants to re-attempt assignments; values progress visualization
- **Tech Level:** High - native mobile user

### Persona 3: Admin (Dr. Patel)
- **Role:** School principal managing multiple classes
- **Pain Points:** Can't easily track teacher workload or identify struggling students across classes
- **Desired Outcome:** System overview, user management, data-driven school insights
- **Behavior:** Focuses on system-level metrics and compliance
- **Tech Level:** Moderate

## How Users Should Experience AI LMS

### Teacher Workflow
1. Open AI LMS → Create assignment with rich questions + rubric
2. Set learning objectives aligned to curriculum
3. Distribute to class with deadline
4. (Auto) AI grades submissions as students submit
5. Review AI grades, override if needed, add comments
6. View class analytics: which students struggle with which concepts
7. Click "Suggest Intervention" → Get AI-recommended strategies
8. Message students with personalized recommendations

### Student Workflow
1. Receive assignment notification
2. Open AI LMS → Start assignment (auto-saves as typing)
3. Upload work if needed (images, PDFs)
4. Review answers before submitting
5. Hit "Submit" → Instant AI feedback appears
6. Read feedback and error explanations
7. (Optional) Re-attempt assignment based on feedback
8. View personal learning dashboard: progress on each skill

### Admin Workflow
1. Dashboard shows school-wide metrics
2. Manage users and classes
3. View teacher workload analytics
4. Monitor high-risk students
5. Export reports for administrative purposes

## User Experience Goals

### Core UX Principles
1. **Instant Feedback Culture** - Students should feel AI evaluating as if teacher is watching
2. **Clarity Over Complexity** - Every feature has a clear purpose; no fluff
3. **Mobile-First** - Teachers grade on phone between classes; students use anywhere
4. **Progress Visualization** - Learners should see improvement trends clearly
5. **Accessibility** - Works for all learning styles; readable fonts, color-blind friendly
6. **Error as Learning** - Mistakes are framed as learning opportunities, not failures

### Design Patterns
- **Progressive Disclosure** - Show essentials first, advanced options on demand
- **Confirmation & Undo** - High-impact actions require confirmation or offer undo
- **Consistent Language** - Vietnamese UI with consistent education terminology
- **Responsive Feedback** - Every action gets immediate feedback (loading states, success messages)

## Competitive Advantages
1. **AI-Powered, Not Replaced** - Teachers remain central, AI augments their work
2. **Instant Turnaround** - No delays in feedback; momentum for learning
3. **Learning Pattern Detection** - Identifies struggling students early
4. **Teacher Control** - Override capability + detailed feedback tuning
5. **Mobile-First** - Works seamlessly on phones (teachers/students)
6. **Learning Objectives Aligned** - Tracks skills, not just grades
